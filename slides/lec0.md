## Part 1. 为什么会有这门课

-v-

### 这门课想补上的是什么

- 你会写一点代码，不等于你已经会高效地学习、排错、协作、表达
- 很多真正反复影响体验的内容，学校课程往往默认你会，但不会系统讲
    - 环境配置
    - 命令行与系统基础
    - 搜索、提问、读文档
    - Git / GitHub 协作
    - 技术写作与排版
    - 与 AI 协作完成实际任务
- 这门课补的不是某一门专业课，而是做事的基础工作流

-v-

### 为什么学校正常课程之外还需要它

> Classes teach you all about advanced topics within CS, from operating systems to machine learning, but there’s one critical subject that’s rarely covered, and is instead left to students to figure out on their own: proficiency with their tools.
>
> <div style="text-align: right;">—— <a href="https://missing-semester-cn.github.io/">MIT《计算机教育中缺失的一课》</a></div>

- 之后几年里，你会花大量时间和工具、环境、文档、版本控制打交道
- 这些能力平时不显山露水，但会持续决定你的效率和心态
- 很多看似“我太菜了”的痛苦，实际上是工作流没有建立起来

-v-

### 历代「技能拾遗」一直在回答同一个问题

- 一个计院学生，除了课堂知识，还需要掌握什么实际能力？
- 历代课程内容不断变化，但主线其实很稳定：
    - Linux / Shell
    - Git / GitHub
    - Markdown / LaTeX / Typst
    - 搜索、提问、排错
    - 更专业的工作方式
- 所以今年的版本也不是推倒重来，而是在 AI 时代重新组织这条主线

-v-

### 什么叫“更像一个 CS 人”

- 不是会背很多术语
- 不是桌面很黑、终端很多窗口就算专业
- 而是逐渐形成这样的习惯：
    - 遇到问题先观察、再定位、再验证
    - 知道去哪里找更可靠的信息
    - 能把自己的环境、代码、文档、过程整理清楚
    - 愿意持续打磨自己的工作流

-v-

### 这门课不是什么

- 不是命令大全背诵课
- 不是工具展览课
- 不是某个 AI 产品的使用说明书
- 也不是一门“听完就会”的速成课

这门课更像是：

- 一门现代计算机工作流入门课
- 一门帮助你减少低级重复痛苦的课
- 一门帮助你更独立、更专业地完成任务的课

## Part 2. AI 时代，为什么这些能力反而更重要

-v-

### 时代确实变了

- 以前大家常说：
    - STFW
    - RTFM
    - RTFSC
- 而今天，很多人的第一反应已经变成了：
    - ATFAI
- 这不是坏事
- 真正的问题是：
    - 你会不会用 AI 学习和工作
    - 你能不能判断它什么时候在胡说
    - 你能不能验证它给出的结果

-v-

### 有 AI 了，为什么还要学这些

- AI 不替你理解上下文
- AI 不替你管理环境
- AI 不替你承担版本一致性
- AI 不替你判断输出是否可信
- AI 不会自动给你一个干净、稳定、可复现的工作流

一句话：

- **AI 放大能力，也放大混乱。**

-v-

### AI 时代更需要“机器永远是对的”

- 程序不会“闹情绪”
- 它只是按机制运行
- 所谓玄学 bug，通常只是：
    - 输入不对
    - 环境不对
    - 版本不对
    - 路径不对
    - 权限不对
    - 假设不对
- 所以后面这门课会不断训练一件事：
    - 相信证据，尊重机制，减少拍脑袋

-v-

### 抽象没有过时，反而更重要

- AI 可以帮你写命令、写代码、解释概念
- 但你还是得知道自己在和什么层次打交道：
    - 文件 / 目录
    - 本地 / 远程
    - 源码 / 构建产物
    - 进程 / 权限
    - 输入 / 输出 / 日志
    - 当前状态 / 历史版本 / diff
- 很多问题本质上不是“不会某条命令”
- 而是抽象层次搞混了

-v-

### 这门课里的 AI 定位

- AI 不会只出现在最后一讲
- AI 会贯穿整门课：
    - 帮你理解概念
    - 帮你搜索与整理信息
    - 帮你排查问题
    - 帮你写文档
    - 帮你写代码和自动化脚本
- 但每次都会强调：
    - 什么时候适合用
    - 什么时候不能直接信
    - 怎么验证
    - 边界在哪里

## Part 3. 这门课会带你走一条怎样的路

-v-

### 一张课程总地图

1. Lec 0：课程前瞻与环境准备
2. Lec 1：学习、搜索、提问与排错
3. Lec 2：Linux 与命令行工作流入门
4. Lec 3：Linux 系统基础与远程开发
5. Lec 4：Git、GitHub 与开源协作
6. Lec 5：Markdown、技术写作与文档表达
7. Lec 6：LaTeX / Typst 与科学写作排版
8. Lec 7：AI 协作编程与 Agent 工作流
9. 可选 Lec 8：网络 / Web 基础或综合实践

-v-

### 这不是“学一堆工具”，而是建立工作流

这门课真正想带你建立的是四类能力：

- 获取信息
    - 会搜索、会提问、会读文档、会借助 AI
- 操作环境
    - 理解终端、系统、远程环境与依赖
- 管理过程
    - 用 Git 管理版本、协作与变更
- 输出结果
    - 用 Markdown / LaTeX / Typst / AI 把内容表达清楚

-v-

### 学完之后，希望你至少得到这些

- 不会再把很多问题都归结成“电脑有问题”
- 能更独立地开始一项新的技术任务
- 能更快地定位问题、求助、验证方案
- 能搭起一个相对干净的本地工作环境
- 能开始使用版本控制、文档和 AI 组成自己的工作流
- 能少掉很多重复、低效、玄学式折腾

-v-

## Part 4. 在开始之前：环境准备、资料入口与生存建议

-v-

### 先认识你的电脑

至少先知道这些问题的答案：

- 你在用什么操作系统
- 你的文件放在哪里
- 你平时用什么终端 / Shell / 编辑器
- 软件一般装在哪里
- PATH 大概在干什么

不是要求你现在全懂，而是先建立意识：

- **电脑不是黑箱。**

-v-

### 准备一个类 Unix 环境

- Linux：直接可用
- macOS：终端天然可用
- Windows：建议优先准备 WSL 或可用的远程 Linux 环境
    - WSL 文档：<https://learn.microsoft.com/zh-cn/windows/wsl/>

原因很简单：

- 后面很多内容在类 Unix 环境里更自然
- 不是平台鄙视，而是现代计算工作流的现实

-v-

### 建议你尽快准备好的基础工具

- GitHub 账号
- 一个可用的终端
- 一个顺手的编辑器 / IDE
- 一个能稳定使用的 AI 工具
- 基本的下载、联网、同步资料能力

不用一次性全都折腾到最好，先能稳定使用即可。

-v-

-v-

### 很快你就会遇到的几类坑

- 命令装了但终端里找不到
- AI 给你的命令复制后不能运行
- 一样的代码，别人能跑你不能跑
- 路径、权限、编码、版本总有一个在捣乱
- 你以为你在改源码，其实你改的是别的文件

这些都非常正常。

后面的课程会教你怎么系统地处理它们。

-v-

### 安全与边界意识

- 不要把 token、密钥、`.env`、私有代码随手贴给 AI
- 不要看到 `sudo` 就直接回车
- 不要复制来路不明的一长串命令直接执行
- 不要把“能跑”误认为“安全、正确、可复现”

未来你越依赖自动化和 AI，这一页越重要。
